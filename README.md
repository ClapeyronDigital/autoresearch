# autoresearch-mnist

Игрушечная версия [autoresearch](https://github.com/karpathy/autoresearch) — AI-агент автономно экспериментирует с нейросетью для классификации MNIST (цифры 0–9), пытаясь выжать максимальную accuracy за фиксированное время.

## Как это работает

### Архитектура — четыре файла

| Файл | Кто редактирует | Содержит |
|---|---|---|
| `prep_mnist.py` | **Никто** (фиксирован) | Загрузка MNIST, даталоадеры, функция оценки |
| `train_mnist.py` | **Агент** | Модель, гиперпараметры, цикл обучения |
| `program_mnist.md` | **Человек** | Инструкции и лимиты для агента |
| `analysis.py` | **Человек** (после агента) | Визуализация прогресса → `progress.png` |

Идея в разделении ответственности:
- **Человек** пишет инструкции агенту (`program_mnist.md`) — что делать, какие правила, как логировать
- **Агент** меняет только `train_mnist.py` — архитектуру, оптимизатор, гиперпараметры, тренировочный цикл
- **Метрика** зафиксирована в `prep_mnist.py` — агент не может её «подкрутить», только улучшить модель

### Цикл работы агента

Агент запускает цикл, каждая итерация — один эксперимент. Количество экспериментов задаётся в `program_md.md` параметром `MAX_EXPERIMENTS` (0 = без лимита):

1. **Придумать гипотезу** — например: «добавить Dropout 0.2», «заменить ReLU на GELU», «переписать MLP на CNN»
2. **Отредактировать `train_mnist.py`** — поменять гиперпараметры или переписать класс `Model`
3. **Сделать git commit** — зафиксировать изменение
4. **Запустить тренировку**: `uv run train_mnist.py > run.log 2>&1`
5. **Извлечь результат**: `grep "^val_accuracy:" run.log`
6. **Записать в `results.tsv`** — строку с хешем коммита, accuracy, статусом и описанием
7. **Принять решение**:
   - **accuracy выросла** → оставить коммит (keep)
   - **accuracy упала или не изменилась** → `git reset --hard HEAD~1` (откатить)
   - **скрипт упал** → откатить, статус «crash», попробовать починить или двигаться дальше
8. **Повторить**, пока не достигнут `MAX_EXPERIMENTS` (если задан)

Агент **никогда не спрашивает разрешения** продолжать — работает автономно до лимита или до ручной остановки.

### Временной бюджет

Каждая тренировка длится **ровно 60 секунд** (wall clock). Это ключевое дизайнерское решение:

- Все эксперименты получают одинаковое количество вычислительного времени
- Результаты сравнимы между собой независимо от архитектуры, batch size и т.д.
- Агент оптимизирует **качество за единицу времени**, а не просто качество
- ~60 экспериментов в час, ~500 за ночь

### Что агент может менять

Всё, что угодно, в `train_mnist.py`:

- **Архитектуру** — количество слоёв, размер скрытого слоя, тип слоёв (Linear, Conv2d, ...)
- **Функции активации** — ReLU, GELU, SiLU, LeakyReLU
- **Оптимизатор** — Adam, SGD, AdamW, ... и их параметры
- **Регуляризацию** — Dropout, BatchNorm, Weight Decay
- **Данные** — данные фиксированы, но агент может добавить аугментацию прямо в цикл обучения (RandomAffine, RandomErasing на тензорах)
- **LR-расписание, gradient clipping, label smoothing, ...**

Агент может **полностью переписать класс `Model`** — с нуля, в любом виде. Единственное требование: `model(x)` принимает батч `(B, 1, 28, 28)` и возвращает логиты `(B, 10)`.

### Что агент НЕ может менять

- `prep_mnist.py` — даталоадер, функция оценки, константы (TIME_BUDGET)
- Зависимости в `pyproject.toml` — только torch, torchvision, matplotlib
- MNIST — 60k train / 10k test, стандартная нормализация

Это гарантирует, что метрика честная и эксперименты воспроизводимы.

## Быстрый старт

```bash
# 1. Установить uv (если ещё нет)
brew install uv              # или: curl -LsSf https://astral.sh/uv/install.sh | sh

# 2. Установить зависимости
uv sync

# 3. Скачать MNIST (один раз, ~12 MB)
uv run prep_mnist.py

# 4. Запустить одну тренировку вручную (~60 сек) — проверить, что всё работает
uv run train_mnist.py

# 5. Настроить лимит экспериментов в program_mnist.md
#    Открой program_mnist.md и поставь нужное число:
#    MAX_EXPERIMENTS = 30    (0 = без лимита)

# 6. Запустить агента (через Claude Code, Codex или другой coding agent)
#    Открыть агента в этой папке и дать промпт:
#    "Hi, have a look at program_mnist.md and let's kick off a new experiment!"

# 7. После завершения — построить график прогресса
uv run analysis.py
#    → progress.png
```

Требования: macOS / Linux / Windows, Python 3.10+. Работает на CPU, MPS (Apple Silicon) и CUDA — устройство определяется автоматически.

## Формат результатов

### progress.png

После завершения экспериментов запусти `uv run analysis.py` — скрипт построит график прогресса:

- X = порядковый номер keep-эксперимента (discard и crash не учитываются)
- Y = val_accuracy
- График сохраняется в `progress.png` рядом с `results.tsv`

### Вывод train_mnist.py

После каждой тренировки скрипт печатает:

```
---
val_accuracy: 0.9776
training_seconds: 60.0
total_seconds: 61.1
peak_memory_mb: 2.2
num_params: 101770
num_steps: 8797
```

Агент извлекает метрику грепом: `grep "^val_accuracy:" run.log`

### results.tsv

Агент ведёт лог экспериментов в `results.tsv` (табы, не запятые). Файл не коммитится в git, лежит локально.

Колонки:

| commit | val_accuracy | memory_mb | status | description |
|---|---|---|---|---|
| a1b2c3d | 0.9120 | 123.4 | keep | baseline MLP |
| b2c3d4e | 0.9340 | 130.2 | keep | added 2nd hidden layer |
| c3d4e5f | 0.9310 | 130.2 | discard | switch to SGD |
| d4e5f6g | 0.0000 | 0.0 | crash | typo in Model init |
| e5f6g7h | 0.9520 | 150.8 | keep | replace MLP with small CNN |

- **keep** — accuracy выросла, коммит оставлен
- **discard** — accuracy не улучшилась, коммит откачен
- **crash** — скрипт упал, коммит откачен

### Git

Эксперименты идут на отдельной ветке `autoresearch/<tag>` (например `autoresearch/may16`). Коммиты в ней — это история успешных (keep) изменений. Неудачные откатываются, в истории не остаются.

## Идеи для развития

Текущий проект — минимальная отправная точка. Возможные направления:

- **Усложнить программу агента** (`program_mnist.md`) — добавить исследовательские стратегии, приоритеты идей, анализ трендов
- **Добавить агентов** — несколько параллельных агентов на разных ветках, обменивающихся идеями
- **Сменить задачу** — CIFAR-10, текстовые задачи, регрессия и т.д.
- **Увеличить бюджет** — 60 сек → 300 сек для более сложных моделей

## Отличия от оригинального autoresearch

| Оригинал (karpathy) | Эта версия |
|---|---|
| LLM (GPT, Flash Attention) | CNN / MLP для MNIST |
| BPE токенизатор, parquet-шарды | torchvision MNIST |
| Метрика: val_bpb (меньше = лучше) | val_accuracy (больше = лучше) |
| 5 минут на эксперимент | 60 секунд |
| H100 GPU | CPU / MPS / CUDA авто-детект |
| ~20 зависимостей | torch + torchvision + matplotlib |
| Модель привязана к структуре GPTConfig | Агент может переписать Model полностью с нуля |
| Бесконечный цикл | Настраиваемый лимит (MAX_EXPERIMENTS) |

Принцип автономного исследования — тот же.
